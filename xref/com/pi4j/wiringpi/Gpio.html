20   * <h1>WiringPi GPIO Control</h1>
21   * 
22   * <p>
23   * Some of the functions in the WiringPi library are designed to mimic those in the Arduino Wiring
24   * system. There are relatively easy to use and should present no problems for anyone used to the
25   * Arduino system, or C programming in-general.
26   * </p>
27   * 
28   * <p>
29   * The main difference is that unlike the Arduino system, the main loop of the program is not
30   * provided for you and you need to write it yourself. This is often desirable in a Linux system
31   * anyway as it can give you access to command-line arguments and so on. See the examples page for
32   * some simple examples and a Makefile to use.
33   * </p>
34   * 
35   * <p>

495      * <p>

531     /**
532      * <h1>[Interrupts]</h1>
533      * 
534      * With a newer kernel patched with the GPIO interrupt handling code, you can now wait for an
535      * interrupt in your program. This frees up the processor to do other tasks while you're waiting
536      * for that interrupt. The GPIO can be set to interrupt on a rising, falling or both edges of
537      * the incoming signal.
538      * 
539      * <p>
540      * <b> int waitForInterrupt (int pin, int timeOut) </b>
541      * </p>
542      * 
543      * <p>
544      * When called, it will wait for an interrupt event to happen on that pin and your program will
545      * be stalled. The timeOut parameter is given in milliseconds, or can be -1 which means to wait
546      * forever.
547      * </p>
548      * 
549      * <p>
550      * Before you call waitForInterrupt, you must first initialize the GPIO pin and at present the
551      * only way to do this is to use the gpio program, either in a script, or using the system()
552      * call from inside your program.
553      * </p>
554      * 
555      * <p>
556      * e.g. We want to wait for a falling-edge interrupt on GPIO pin 0, so to setup the hardware, we
557      * need to run:
558      * 
559      * <pre>
560      * gpio edge 0 falling
561      * </pre>
562      * 
563      * </p>
564      * 
565      * @see <a
566      *      href="https://projects.drogon.net/raspberry-pi/wiringpi/functions/">https://projects.drogon.net/raspberry-pi/wiringpi/functions/</a>
567      * @param pin <p>
568      *            The GPIO pin number. </br><i>(Depending on how wiringPi was initialized, this may
569      *            be the wiringPi pin number or the Broadcom GPIO pin number.)</i>
570      *            </p>
571      * @param timeout <p>
572      *            The number of milliseconds to wait before timing out. </br>A value of '-1' will
573      *            disable the timeout.
574      *            </p>
575      * @return <p>
576      *         The return value is -1 if an error occurred (and errno will be set appropriately), 0
577      *         if it timed out, or 1 on a successful interrupt event.
578      *         </p>
579      */
580     public static native int waitForInterrupt(int pin, int timeout);
