33  /**
34   * <h1>WiringPi LCD Library</h1>
35   * 
36   * <p>
37   * Part of wiringPi is a library to allow access to parallel interface LCD displays (Those that use
38   * the popular Hitachi HD44780U or compatible controllers)
39   * </p>
40   * 
41   * <p>
42   * The library is simple to use in your own programs, however wiring the displays up may be
43   * challenging, so do take care. It is possible to wire up more than one display! In 8-bit mode, the
44   * first display needs 10 GPIO pins and each additional display needs just one more pin, so with a
45   * maximum of 17 GPIO pins, that's 8 displays. If you move to using a 4-bit interface (trivial in
46   * the code), then it's 4 more displays and 12 LCDs! However I suspect the rest of the wiring might be
47   * somewhat challenging. Wiring is described at the end of the this page.
48   * </p>
49   * 
50   * <p>
51   * The LCD display can be either a 5V display or a 3,3v display, however if we are using a 5V
52   * display then we must make absolutely sure the display can never write data back to the Raspberry
53   * Pi, otherwise it will present 5V on the Pi's GPIO pins which will not be good. At best you'll
54   * destroy the pin drivers, at worst you'll destroy your Pi.
55   * </p>
56   * 
57   * <p>
58   * So make sure you always connect the R/W pin on the display to ground to force the display to be
59   * read-only to the host.
60   * </p>
61   * 
62   * <p>
63   * Before using the Pi4J library, you need to ensure that the Java VM in configured with access to
64   * the following system libraries:
65   * <ul>
66   * <li>pi4j</li>
67   * <li>wiringPi</li>
68   * </ul>
69   * <blockquote> This library depends on the wiringPi native system library.</br> (developed by
70   * Gordon Henderson @ <a href="https://projects.drogon.net/">https://projects.drogon.net/</a>)
71   * </blockquote>
72   * </p>
73   * 
74   * @see <a href="http://www.pi4j.com/">http://www.pi4j.com/</a>
75   * @see <a
76   *      href="https://projects.drogon.net/raspberry-pi/wiringpi/lcd-library/">https://projects.drogon.net/raspberry-pi/wiringpi/lcd-library/</a>
77   * @author Robert Savage (<a
78   *         href="http://www.savagehomeautomation.com">http://www.savagehomeautomation.com</a>)
79   */

88      /**
89       * <p>
90       * First, you need to initialize wiringPi in the way you want to. The LCD library will call
91       * pinMode functions, but these are ignored if you have already set the modes using the gpio
92       * program and want to use the wiringPiSetupSys() mechanism.
93       * </p>
94       * 
95       * <pre>
96       * int lcdInit(int rows, int cols, int bits, int rs, int strb, int d0, int d1, int d2, int d3, int d4,
97       *         int d5, int d6, int d7);
98       * </pre>
99       * 
100      * <p>
101      * This is the main initialization function and must be called before you use any other LCD
102      * functions.
103      * </p>
104      * 
105      * <p>
106      * Rows and cols are the rows and columns on the display (e.g. 2, 16 or 4,20). Bits is the
107      * number of bits wide on the interface (4 or 8). The rs and strb represent the pin numbers of
108      * the displays RS pin and Strobe (E) pin. The parameters d0 through d7 are the pin numbers of
109      * the 8 data pins connected from the Pi to the display. Only the first 4 are used if you are
110      * running the display in 4-bit mode.
111      * </p>
112      * 
113      * <p>
114      * The pin numbers will be either wiringPi pin numbers of GPIO pin numbers depending on which
115      * wiringPiSetup function you used.
116      * </p>
117      * 
118      * <p>
119      * The return value is the handle to be used for all subsequent calls to the lcd library when
120      * dealing with that LCD, or -1 to indicate a fault. (Usually incorrect parameters)
121      * </p>
122      * 
123      * @see <a
124      *      href="https://projects.drogon.net/raspberry-pi/wiringpi/lcd-library/">https://projects.drogon.net/raspberry-pi/wiringpi/lcd-library/</a>
125      * 
126      * @param rows
127      * @param cols
128      * @param bits
129      * @param rs
130      * @param strb
131      * @param d0
132      * @param d1
133      * @param d2
134      * @param d3
135      * @param d4
136      * @param d5
137      * @param d6
138      * @param d7
139      * @return return value
140      */
